# Phases 182–184 — Assisted Exit, Shadow Research, Execution Console

Builds on [Phase 181](./PHASE_181.md). **No live auto exits. No auto trims. Human supervision required.**

## Phase 182 — Assisted Exit Intelligence

### Architecture

```
Paper execution → lifecycle state → post-entry telemetry → exit advisories → human exit
```

### Module

`frontend/src/app/services/assisted-exit-intelligence/`

| File | Role |
|------|------|
| `assisted-exit.models.ts` | States, advisories, metrics |
| `continuation-survival.engine.ts` | Continuation health / survival |
| `persistence-hold.engine.ts` | Hold quality, persistence survival |
| `exit-pressure.engine.ts` | Exit pressure score |
| `trailing-quality.engine.ts` | Trailing / MFE capture |
| `second-leg-monitor.engine.ts` | Second-leg probability |
| `exit-advisory.engine.ts` | Advisory hierarchy |
| `assisted-exit.service.ts` | Orchestration + learning stats |

### Lifecycle states

`ENTRY_ACTIVE` → `PERSISTING` → `TRAILING_CONTINUATION` / `SECOND_LEG_ACTIVE` → `REDUCE_RISK` → `EXIT_WARNING` → `EXIT_CRITICAL` → `FAILED_CONTINUATION`

### Exit advisory hierarchy (priority desc)

1. `EXIT_STRUCTURE_FAILURE` / `FAILED_CONTINUATION`
2. `PERSISTENCE_WEAKENING`
3. `VWAP_FAILURE_RISK` / `EXHAUSTION_RISING`
4. `REDUCE_INTO_EXTENSION`
5. `SECOND_LEG_ACTIVE` / `TRAIL_CONTINUATION` / `HOLD_PERSISTENCE`

### UI (Execution Monitor)

Hold quality, continuation health, second-leg probability, exit pressure, persistence survival, post-entry MFE, learning stats (premature exit, missed continuation, etc.).

## Phase 183 — Autonomous Exit Research (shadow only)

### Architecture

```
Paper position → parallel shadow models → simulated paths → ranking → regime insights
```

### Module

`frontend/src/app/services/autonomous-exit-research/`

Shadow models: `LEGACY_RR`, `AUTONOMOUS_TEMPLATE`, `PERSISTENCE_TRAIL`, `SECOND_LEG_HOLD`, `VWAP_PERSISTENCE_EXIT`, `EARLY_EXHAUSTION_EXIT`, `ADAPTIVE_TRAIL`

**Primary metric:** continuation monetization efficiency (not win rate alone).

### Page

`/exit-research` — model rankings, survival curve, second-leg capture, regime-specific winners.

## Phase 184 — Execution Console

### Architecture

```
Scanner (dashboard feed) → paper entries → Execution Console → advisories → manual exit → analytics
```

### Page

`/execution-console` — operational center (10 sections): dominant opportunities, active executions, exit advisories, persistence, continuation, P&amp;L, quality, regime performance, second-leg, shadow insights.

### Performance

5s poll on monitor API only; feed read from existing `RealTimeExecutionService` snapshot; no replay rescoring.
