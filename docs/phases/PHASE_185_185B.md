# Phases 185 / 185B — Lightweight Live Apps

Separate from the Angular research platform (`frontend/` on port **4300**).

## Apps

| App | Folder | Port | Purpose |
|-----|--------|------|---------|
| Research | `frontend/` | 4300 | Replay, analytics, execution research |
| Live Trader | `frontend-live-trader/` | **4400** | Full operational terminal (185B) |
| Live Screener | `frontend-live-screener/` | **4410** | Screener-only (185) |

Backend (shared): **8180** (`evolution` profile)

## Start

```bash
./start-evolution.sh      # or ./start-research.sh — backend + Angular 4300
./start-live-trader.sh    # operational React 4400
./start-live-screener.sh  # screener-only React 4410
```

## Shared API (`/api/live-trader`)

| Endpoint | Poll tier | Description |
|----------|-----------|-------------|
| `GET /tier1` | 1s | Dominant + top 8 ranked |
| `GET /snapshot` | 5s | P&L, positions, advisories, market |
| `PUT /runtime` | — | SCAN / TELEGRAM / AUTO EXEC toggles |

Reuses (no duplicate scanner logic):

- `RealTimeExecutionEngine` feed
- `IntelligenceSnapshotService.scannerOpportunities`
- `MarketHeartbeatService`
- `PaperExecutionResearchService`
- `TelegramAlertService` (cooldowns in `LiveTraderTelegramService`)

## Runtime controls

- **SCAN ON/OFF** — pauses tier1/snapshot polling server-side
- **TELEGRAM ON/OFF** — deduped alerts (DOMINANT_NOW, EMERGING_FAST, etc.)
- **AUTO EXEC** — `PAPER_RESEARCH` only; 1-share probes via existing paper API
